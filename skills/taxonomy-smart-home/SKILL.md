# 智能家居行业词典 Skill（taxonomy-smart-home）

本 Skill 提供两类词典，供其他 Skills 引用：

1. 员工岗位类别（用于把“我是做什么的/我要做什么”映射到对应岗位 Skill）
2. 产品品类与常用俗称（用于把“产品名/设备名/口语表达”映射为可检索的产品关键词）

输出规范：所有输出文本不使用 emoji。

## 一、员工岗位类别（2026 常见）

以下为行业常见岗位分工（来源：行业招聘与全屋智能交付流程资料汇总，2026）：

- 销售类：
  - 销售顾问 / 销售经理 / 销售总监
  - 渠道销售 / 门店导购
  - 售前方案顾问（偏“需求沟通 + 方案设计 + 报价”）
- 交付与项目类：
  - 项目经理（全屋智能项目交付：设计交底、进度质量、验收、协调）
  - 交付工程师 / 系统集成工程师
- 技术施工类：
  - 安装工程师（弱电/布线/设备安装）
  - 调试工程师（网络与系统联动调试、场景配置）
  - 售后工程师（故障排查、上门维护、返修）
- 运营与支撑类：
  - 仓库/物料管理员（出入库、领料、盘点）
  - 财务（收款核验、对账、开票）
  - 人事（入离职、岗位/提成规则变更）
  - 客服（工单受理、回访、派单）

## 三、通用消歧策略（用于减少 token）

当用户输入模糊时，按以下最省 token 的策略处理：

1. 先做“岗位意图分类”：销售/技术/查询工资/管理引导
2. 再做“必要字段最小集”确认：
   - 销售上报最小集：项目、金额、折扣（可默认）、时间（可默认）
   - 技术上报最小集：项目、产品关键词、类型（安装/调试/售后）、数量
   - 工资查询最小集：月份（可默认）
3. 对项目/产品名称：
   - 先用用户原话关键词去 `GET /api/projects`、`GET /api/products` 做模糊匹配（由调用方 Skill 实现）
   - 若候选 <= 3 个，只返回候选列表并让用户用序号确认（最省 token）
   - 若候选很多，提示用户补充更具体的一个词（例如“开关型号/房间/项目简称”）
