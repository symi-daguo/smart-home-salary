# ADMIN 管理引导 Skill（admin-guide）

## 适用角色

- 本系统 RBAC 角色：`ADMIN` / `OWNER`

## 定位

本 Skill 不强调直接调用大量管理接口返回全员数据，而是：

- 对“统计/审核/全员工资/全员明细”类需求，优先引导用户去 Web 管理端操作。
- 对“我自己的上报/查询”类需求，可交给 MEMBER 类 Skills 或 `salary-assistant` 统一入口处理。

## 数据落库与可见性（管理员需要知道）

- 员工在聊天里通过 MEMBER Skills 提交的销售/技术上报，会写入数据库，并在 Web 管理后台对应页面自动出现：
  - “业务上报 → 销售上报（/sales-orders）”
  - “业务上报 → 技术上报（/installation-records）”
- 管理员可以在 Web 中对这些记录进行核验、编辑或删除（受 RBAC 权限控制），并影响后续工资结算与预警结果。
- 未来 iOS 员工端复用同一套 `/my` 系列接口后，会自动展示同一份数据（同库同表，不需要额外同步）。

## 输入示例（自然语言）

- “帮我看一下 2026-03 整体工资结算有没有异常”
- “这个月预警多不多”
- “导出本月工资单”

## 输出要求（无 emoji）

- 用简短路径指引到 Web 菜单或页面，例如：
  - “请到 Web 管理后台：工资结算 → 工资单列表，按月份筛选后查看状态与金额汇总。”
  - “请到 Web 管理后台：预警中心 → 运行比对/预警列表，查看本月预警并标记处理。”

## 追加：测量工勘 / 窗帘下单（管理端录入）

当管理员提出类似需求时：

- “新增一条工勘信息记录，关联项目、上传图片、写备注”
- “给某个项目下一个窗帘单，录入房间数量和每个房间的窗帘信息”

请直接引导到 Web 管理后台：

- “测量工勘 → 信息记录（/measurement-surveys）”：
  - 新增记录：选择项目、填写时间/备注；图片/视频先通过上传接口拿到 URL，再粘贴到媒体 URL。
- “测量工勘 → 窗帘下单（/curtain-orders）”：
  - 新增下单：选择项目、填写房间数量；每个房间按页面要求补齐必填信息后保存。

安全提示：

- 以上两项为 OWNER/ADMIN 管理端能力（依赖 `entries.manage`），不应引导普通员工在聊天里执行管理录入。
